Introduction

  Depth perception is an essential for visually-impaired people. Given an object in front of the viewer, one may not be able to infer distance purely from the visuals. Limited depth perception can therefore be compensated for by a depth map, providing the relative distance to the object projected on each pixel of an image from a camera feed. An application for real-time depth estimation would take in a camera feed and provide a visual indication on top of the 2D image. Alternatively, for the example of navigation of a blind individual, an auditory indication could be used to indicate the distance to the nearest large object detected by the sensor. For a low-vision individual, a depth estimation model could be combined with magnification to estimata the distance of faraway objects.

Depth Maps

This is an example of a depth map. The darker pixels represent farther away parts of the image while the lighter pixels represent closeness. In this example, the depth values themselves are relative. In metric depth estimation, specific distance values are assigned to each pixel. For example, a particular pixel may be a distance of 2 meters away from the viewer. In this example, the drink itself is nearby while the background, indicated by the darker colors, is farther away. Additionally, the cup in the background has a middling shade to represent it as further from the drink but closer than the background.

The following button generates a random depth map using noise. The darker values are used to represent parts of the image that are further away while the lighter parts indicate nearby objects.

  The goal of depth estimation is to take in input from a camera and create the depth map as explained above. In an accessibility application, for example, the software handles a live feed and calculates the depth map, displaying it in real-time.
